Fils de syndication

Vous cherchez la BOEW v3.1?

À partir du 23 septembre 2014, la version 3.1 de la Boîte à outils de l'expérience Web n'est plus supportée. Le code source et la documentation ont été déplacés vers le dépôt wet-boew-legacy.

Questions ou commentaires?

But

Le but de cette fonctionnalité est de permettre de regrouper et afficher les articles d'un ou plus fils de syndication sur une page Web. Les formats de fil de syndication supportés sont les formats Atom, RSS, et Media RSS.

Utiliser lorsque

  • Pour faire la promotion ou le partage d'informations provenant de fils de syncation sur une page Web.

Mise en œuvre

  1. Pour chaque région qui affichera des fils de syndication, ajouter un élément de type section portant l'attribut class="wb-feeds". 
    <section class="wb-feeds"></section>
  2. À l'intérieur de l'élément de type section, ajouter un titre (h1 - h6).
  3. Ajouter un liste non ordonnée (ul) portant l'attribut class="feeds-cont" après le titre.
    • Fils Facebook :
      1. Ajouter l'attribut class="media-list" à la liste non ordonnée.
      2. Ajouter le code <div class="clearfix"></div> après la liste non ordonnée.
    • Fils Flickr et YouTube :
      1. Ajouter l'attribut class="list-inline" à la liste non ordonnée.
      2. Ajouter le code <div class="clearfix"></div> après la liste non ordonnée.
  4. Pour chaque fil de syndication à afficher, ajouter un élément de type liste contenant un lien vers le fil de syndication.
    • Fils de syndication normaux : 
      <li>
	<a href="http://canada.ca/sample.atom">Sample Atom Feed</a>
</li>
    • Fils Facebook :
      1. Ajouter l'attribut class="media" à l'élément de type liste.
      2. Ajouter l'attribut class="pull-left" au lien.
      3. Ajouter une image de profil utilisateur ayant l'attribut class="media-object" au lien.
      4. Ajouter un élément de type section ayant l'attribute class="media-body" après le lien.
      5. À l'intérieur de l'élément de type section, ajouter un titre (h1 - h6) ayant l'attribut class="media-heading". Include le nom du compte Facebook au titre (e.g., "Parcs Canada - Facebook").
      6. Ajouter un élément de type p contenant la description du compte Facebook après le titre.
      <li class="media">
	<a class="pull-left" href="https://www.facebook.com/feeds/page.php?id=137563459593611&amp;format=atom10">
		<img class="media-object" src="https://fbcdn-profile-a.akamaihd.net/hprofile-ak-ash3/t1.0-1/p320x320/10245484_872682866081663_8340097330550277155_n.png" alt="Parks Canada Logo" width="64" />
	</a>
	<section class="media-body">
		<h4 class="media-heading"><a href="https://www.facebook.com/feeds/page.php?id=137563459593611&amp;format=atom10" rel="external">Parks Canada - Facebook</a></h4>
		<p>On behalf of the people of Canada, we protect and present nationally significant examples of Canada's natural and cultural heritage, and foster public understanding, appreciation and enjoyment in ways that ensure the ecological and commemorative integrity of these places for present and future generations.</p>
	</section>
</li>
    • Fils Flickr :
      1. L'attribut href du lien doit contenir l'URL du fil de photos du compte Flickr.
      2. Ajouter un attribut data-ajax contenant l'URL du fil JSON du compte Flickr au lien.
      <li>
	<a href="https://www.flickr.com/photos/environmentcan" rel="external" data-ajax="https://api.flickr.com/services/feeds/photos_public.gne?id=47711201@N05&amp;format=json">Environment Canada Flickr</a>
</li>
    • Fils YouTube :
      1. L'attribut href du lien doit contenir l'URL du compte YouTube.
      2. Ajouter un attribut data-ajax contenant l'URL du fil JSON du compte YouTube au lien.
      <li>
	<a href="https://www.youtube.com/user/ParksCanadaAgency" rel="external" data-ajax="https://gdata.youtube.com/feeds/api/users/UCzJRhTw3KVtDlcdNz33eDMQ/uploads?v=2&amp;alt=json">Parks Canada Youtube</a>
</li>
  5. Optionnel : Regrouper plusieurs types de fils à l'aide d'une interface à onglets. L'avantage de cette approche est que seuls les fils visibles seront chargés (améliore grandement les performances lorsque beaucoup de fils sont présent).
    1. Ajouter un élément de type div portant l'attribut class="wb-tabs".
    2. Ajouter un élément de type div portant l'attribut class="tabpanels" à l'intérieur du premier élément de type div.
    3. Pour chaque groupe de fils (ils devraient être groupés par type), ajouter un élément de type details. Ajouter l'attribut open="open" au groupe de fils qui devrait être affiché par défaut.
    4. Ajouter un élément de type summary au début de chaque élément de type details contenant le nom du groupe de fils (par exemple, <summary>Facebook</summary>).
    5. Ajouter les régions dans lesquelles afficher les fils de syndication aprés chaque élément de type summary tel que décrit aux étapes 1 à 4, à l'exception d'ajouter l'attribut class="wb-inv" au titre de chaque région.
    <section>
	<h3>Tabbed Social Feeds</h3>
	<div class="wb-tabs col-md-5 wb-eqht">
		<div class="tabpanels">
			<details id="details-facebook" open="open">
				<summary>Facebook</summary>
				<section class="wb-feeds display-5">
					<h4 class="wb-inv">Facebook</h4>
					<ul class="feeds-cont media-list">
						...
					</ul>
					<div class="clearfix"></div>
				</section>
			</details>

			<details id="details-flickr">
				<summary>Flickr</summary>
				<section class="wb-feeds display-10">
					<h4 class="wb-inv">Flickr</h4>
					<ul class="feeds-cont list-inline">
						...
					</ul>
					<div class="clearfix"></div>
				</section>
			</details>

			<details id="details-youtube">
				<summary>YouTube</summary>
				<section class="wb-feeds display-10">
					<h4 class="wb-inv">YouTube</h4>
					<ul class="feeds-cont list-inline">
						...
					</ul>
					<div class="clearfix"></div>
				</section>
			</details>
		</div>
	</div>
	<div class="clearfix"></div>
</section>

Options de configuration

Documente les options de configuration mises à la disposition des intégrateurs et des développeurs.

Option Description Comment la configurer Valeurs
load-x Contrôle la quantité d'articles que le gadget des fils de syndication chargera. Ajouter la classe load-x après la classe wb-feeds et remplacer le x par un nombre entier.
Omis :
Prends la même valeur que l'option display-x ou, si l'option display-x est omise aussi, la valeur par défaut pour le fil de syndication.
load-n :
Limite la quantité d'articles chargés du fil à n articles.
display-x Contrôle la quantité d'articles que le gadget des fils de syndication affichera. Ajouter la classe display-x après la classe wb-feeds et remplacer le x par un nombre entier.
Omis :
Prends la même valeur que l'option load-x ou, si l'option load-x est omise aussi, la valeur par défaut pour le fil de syndication.
display-n :
Limite la quantité d'articles affichés à n articles.

Événements

Documente les événements mis à la disposition des intégrateurs et des développeurs.

Événement Déclancheur Effet
wb-init.wb-feeds Déclanché manuellement (par exemple, $( ".wb-feeds" ).trigger( "wb-init.wb-feeds" );). Sert à initialiser manuellement le gadget des fils de syndication. Nota : Le gadget des fils de syndication sera initialisé automatiquement à moins qu'il soit ajouté à la page après que la page ait été chargée.
wb-ready.wb-feeds (v4.0.5+) Déclanché automatiquement une fois que le plugiciel des fils de syndication ait été initialisé. Sert à identifier là où le plugiciel des fils de syndication a terminé l'initialisation (cible de l'événement). 
$( document ).on( "wb-ready.wb-feeds", ".wb-feeds", function( event ) {
});
$( ".wb-feeds" ).on( "wb-ready.wb-feeds", function( event ) {
});
wb-feed-ready.wb-feeds (v4.0.5+) Déclanché automatiquement une fois qu'un fil de syndication est affiché. Sert à identifier le fil qui a été affiché (cible de l'événement). 
$( document ).on( "wb-feed-ready.wb-feeds", ".wb-feeds .feeds-cont",  function( event ) {
});
$( ".wb-feeds .feeds-cont" ).on( "wb-feed-ready.wb-feeds", function( event ) {
});
wb-ready.wb (v4.0.5+) Déclanché automatiquement lors que la Boîte à outils de l'expérience Web a terminé son chargement et son éxécusion. Sert à identifier le moment où tous les plugiciels et les correctifs ont terminé leur chargement et leur éxécusion. 
$( document ).on( "wb-ready.wb", function( event ) {
});
